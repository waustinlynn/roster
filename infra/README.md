# Infrastructure

All AWS infrastructure for the Roster application, managed entirely through Terraform.

## What's Provisioned

| Component | AWS Service | Notes |
|-----------|-------------|-------|
| Networking | VPC, Subnets, IGW, Route Tables | 2 public + 2 private subnets in `us-east-1` |
| VPC Endpoints | S3 Gateway, ECR API/DKR, CloudWatch Logs | Eliminates NAT gateway for API task |
| Event Store | S3 | Redpanda Tiered Storage archive; Glacier after 90 days |
| ECS Cluster | ECS Fargate | No EC2 to manage |
| API Image Repo | ECR | `roster-api`; immutable tags; scan on push |
| API Service | ECS Fargate (private subnet) | ASP.NET Core; ALB with HTTPS health-check gating |
| Load Balancer | ALB | HTTPS:443 (self-signed cert); HTTP:80 → HTTPS redirect |
| Redpanda Service | ECS Fargate (public subnet) | Single-node event store; pulled from Docker Hub |
| Redpanda Storage | EFS | Hot log segments survive container restarts |
| UI Hosting | S3 + CloudFront | SPA; HTTPS enforced; OAC; SPA 404→200 routing |
| IAM | IAM Roles + Policies | Least-privilege; Redpanda has S3 write; API does not |
| Remote State | S3 + DynamoDB | Manually bootstrapped (see `bootstrap/README.md`) |

## Network Architecture

```
Internet
   │
   ├─ HTTPS/HTTP ──► ALB (public subnets) ──► API Task (private subnet)
   │                                               │ port 9092
   └─ Docker Hub ──► Redpanda Task (public subnet, assign_public_ip=true)
                            │ EFS NFS
                     EFS Mount Targets (private subnets)

Private subnets reach ECR / S3 / CloudWatch via VPC endpoints (no NAT gateway).
```

## Quick Start

### Prerequisites

1. AWS credentials configured (`aws configure` or environment variables)
2. Terraform ≥ 1.6 installed
3. Bootstrap state backend created — see [`bootstrap/README.md`](bootstrap/README.md)
4. API image pushed to ECR with a pinned tag (e.g., `1.0.0`)
5. Redpanda image version decided (e.g., `v23.3.1`) — pulled from Docker Hub at deploy time

### Deploy Dev Environment

```bash
cd infra
# Initialize once (or when switching environments)
terraform init -backend-config=envs/dev.backend.hcl

terraform plan  -var-file=envs/dev.tfvars -var="api_image_tag=1.0.0" -var="redpanda_image_tag=v23.3.1"
terraform apply -var-file=envs/dev.tfvars -var="api_image_tag=1.0.0" -var="redpanda_image_tag=v23.3.1"
```

### Switch to Prod Environment

```bash
terraform init -reconfigure -backend-config=envs/prod.backend.hcl
terraform plan  -var-file=envs/prod.tfvars -var="api_image_tag=1.0.0" -var="redpanda_image_tag=v23.3.1"
terraform apply -var-file=envs/prod.tfvars -var="api_image_tag=1.0.0" -var="redpanda_image_tag=v23.3.1"
```

### Key Outputs

After `terraform apply` succeeds:

| Output | Description |
|--------|-------------|
| `api_url` | ALB DNS name — API is served over HTTPS here |
| `ui_url` | CloudFront domain — SPA is served over HTTPS here |
| `ecr_api_repo` | ECR URI for pushing API images |
| `redpanda_endpoint` | Internal DNS for Redpanda (used by API task) |

### Verify

```bash
# API health check (self-signed cert — use -k)
curl -k https://<api_url>/health

# UI
curl https://<ui_url>/index.html
```

## Security Notes

- The API ALB uses a **self-signed TLS certificate** generated by Terraform. This is acceptable for v1. Replace with an ACM-managed cert when a custom domain is added.
- The Redpanda task has a **public IP** (required for Docker Hub pull) but its security group blocks all inbound traffic from the internet. Kafka port 9092 is only reachable from within the VPC.
- All Terraform variables for image tags **have no default** — you must supply them explicitly to prevent accidental `latest` deployments.
