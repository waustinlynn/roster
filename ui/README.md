# UI — Roster Management SPA

React + Vite single-page application for the youth sports roster management system.

## What's Here

| Directory | Purpose |
|-----------|---------|
| `src/api/` | Auto-generated OpenAPI client — **do not hand-edit** |
| `src/components/` | Reusable UI components |
| `src/hooks/` | TanStack Query hooks wrapping the generated API client |
| `src/pages/` | Route-level page components |

## Quick Start

```bash
# Install dependencies
npm install --prefix ui

# Start dev server (proxies API calls to localhost)
npm run dev --prefix ui

# Regenerate the API client from the latest OpenAPI spec
npm run generate-client --prefix ui

# Run tests
npm test --prefix ui

# Build for production
npm run build --prefix ui
```

## API Client

The client in `src/api/` is generated by [orval](https://orval.dev) from the OpenAPI spec
exported by the `Roster.Api` project. **Never hand-edit files in this directory.**

To regenerate after API changes:

```bash
# 1. Export the latest spec from the running API
curl http://localhost:5000/swagger/v1/swagger.json > openapi.json

# 2. Regenerate the client
npm run generate-client --prefix ui
```

## Data Fetching

All server state goes through **TanStack Query** hooks in `src/hooks/`. Direct `fetch` or
`axios` calls to the Roster API outside of TanStack Query hooks are not permitted.

```typescript
// Good — use a generated hook wrapped in TanStack Query
import { useGetRosters } from '../hooks/useRosters'
const { data, isLoading } = useGetRosters()

// Bad — bare fetch call
const data = await fetch('/api/rosters').then(r => r.json())
```

## Deployment

The production build (`dist/`) is synced to the S3 bucket provisioned by `infra/modules/ui-hosting`
and served via CloudFront over HTTPS.

```bash
# Build
npm run build --prefix ui

# Deploy (requires AWS credentials + CloudFront distribution ID from Terraform outputs)
aws s3 sync ui/dist/ s3://<s3_bucket_name>/
aws cloudfront create-invalidation --distribution-id <dist_id> --paths "/*"
```
